Zo ver heb ik al mijn Website opdrachten af met weinig moeite.

1e officiële opdracht: Flexbox
Het doel van het Flexbox opdracht was dat je leert hoe je werkt op verschillende manieren met Flexbox.
Je gaat je verdiepen in het opbouwen van een website en leert visueel onderscheidt te houden tussen Flexbox & Grid.

2e officiële opdracht: Componenten library
Componenten library opdrachten was bedoeld dat jij je code vrijgaf aan de "bezoeker" mocht hij/zij je code willen kopiëren.
het ging over je HTML en je CSS code. Er zit verschil tussen het werd duidelijk aangegeven dat de CSS code downloadable moest zijn.

3e officiële opdracht: Responsive opdracht (Framework en/of MediaQueries)
Het doel was dat je ervoor zorgt dat je website Responsive is, dat houd in dat je website zich aanpast aan het device waar jij de website op bezoekt.

Werken met flexbox is op het moment wat ik doe tijdens het maken van deze opdracht. dus je kunt zien dat alles horizontaal is.
Flexbox is het makkelijkste lay-out voor een website. je werkt van links naar rechts net zoals we lezen.
Mijn persoonlijke ervaring ermee is eenvoudig. het was snel op te pakken en makkelijk te leren en mezelf erin te verdiepen.
momenteel mijn favorieten manier om een website mee in te richten.

Mijn eindopdracht "Componenten library" was wat lastiger niet dat het per se moeilijk in te richten maar voornamelijk dat ik geen inspiratie had voor de website zelf.
Ik had er voor visie voor dus wist niet waar ik moest beginnen, dus ik bleef maar coderen en coderen tot ik met iets kwam dat niet optimaal was maar wel voldoet aan de eisen van mijn opdracht.

Het maken van een Responsive website klonk moeilijk, voornamelijk omdat ik niet wist hoe ik moest starten. Ik had voorheen een project gestart waarin in dat heb gedaan, maar dat was al een tijdje gelden.
Een snelle google search heeft me daarbij geholpen en heb mijn opdracht ingeleverd zoals het hoorden.

Nu ik terug kijk op mijn manier van aanpak heb ik best veel geleerd van mijn fouten.
Mijn grootste fout is dat tijdens het werken aan een website ik meestal start zonder een beeld ervan te hebben, dus nu begin ik niet aan een project zonder dat ik echt een beeld erbij heb.
Dat heeft zo zijn na als zijn voordelen.
Ik werk nu veel meer gestructureerd en gefocust.
Mijn doel is het behouden van mijn werkproces en het volbrengen van mijn opdrachten.

Mijn verbeterpunten waar ik nog aan mag werken: sneller werken, het valt me dat ik nogal blijf haken op sommige punten bij het maken van een site bijvoorbeeld het schrijven van informatieve tekst.

Mappenstructuur netjes houden en niet zoveel tegelijkertijd willen doen.

alles opslaan op een Onedrive en niet opslaan op een lokale driver. soms wil het zo zijn dat ik mijn opslag vergeet en dan kan ik niet meer verder werken aan mijn projecten waardoor ik niks af krijg.
